Directory Image Resizer Example

The directory image resizer provides a simple way to to resize images, both local and remote, with clean inline commands and tidy URLs. But showing is easier than explaining, so lets start with this image of a somewhat disgruntled monkey.

Maybe he just doesn't like having his picture plaster so big on a website. I know, lets make it smaller, say, 80px by 80px, that would work will for an avatar or something.

Excellent. But sometimes the image is only a little bit off the dimensions you want, and a little bit of distortion isn't a big deal right? We have a "fuzziness-factor" programmed in for that. it defaults to 10% but can be adjusted in the resize script. Lets see how that works, we have two examples below.

Little bit Taller


Little bit Wider

Those are both just a tad under 10% larger one direction than the other, by default the "fuzziness-factor" is 11%, so when we try and turn them into perfect squares they will get distorted to do so, it's just a little bit, so it's hardly noticeable.

No Longer Wider

You can also use the images original size as a value when giving the image a new size. This is good for things like banners, lets say you want a banner thans 500px wide, but don't really care about the height. can pass it a value like 500xan where a is the automatically calculated height of the image and n sets it to the nearest image size possible. This will resize the iamge to be 500px wide and whatever that would need to be in height in order to contain the whole thing. Lets use this image as an example

./images/crossroad.jpg, 1000px by 562px

We can turn it into a 500px by whatever it needs to be banner like this

./images/500xan/crossroad.jpg

Passing the height as a (for auto) and setting to output the image using the nearest resizing method we created a banner of variable height as needed. You can then give thid iamge a width alone letting the image resize vertically as needed (and horizntally in the case of HiDPI support). You can also replace the vertical size with a for the same effect vertically.

./images/ax250n/crossroad.jpg

If you looking for a large backgroud image you can also use the letter "f" for full-screen sizing. This will use the users screen dismensions. This is used as opposed to the viewport dimensions so that if the browser window is resized the image will still look as shark and as crisp as possible. In addition to resizing however you like Directory Image Resizer also caches resized images to save time. The cache image will always be loaded unless it's source has changed, which is checked with a simple MD5 comparison. Other features include support for HiDpi Devices with a tiny bit of javascript to get the users Pixel Ratio. One thing to remember is that if you are using the retina support your going to have to give your images a width and/or height so they don't go all out of proportion and mess things up, but your doing that already right?
